使用Hexo建立博客网站

发表于 2016-07-30

很早之前就想建立自己的博客网站，因为发现写东西是多数牛人不断进步的共同方法，分享给别人其实是在梳理自己。所以也利用WordPress建立了一个网站，并且还在GoDaddy上申请了域名梦想照进现实,但是写了第一篇后就没有坚持下来，当时的那篇也可以转载过来。原因还是由于写不出来，写不出来的原因按照李笑来的说法有输入太少，没有处理能力。

很早就有搭建Blog站点的想法了，都因自己的惰性和粘性从未开始， 虽然也零零碎碎的在自己的机器上记下了近千篇称之为随记的东西，但始终没有一篇公开的日志。每每看到别人的东西，或谈理想，或谈技术，一切都显得风清云淡，就感慨如果当初自己能够做到并坚持下来，也算是早一批的Blogger了。

小时候不喜欢写日记，坐在那里挤牙膏似的修改着每一个字词，完成老师或家长布置的不喜欢的任务。现在觉得写文章是一件自然的事情，脑子每时每刻都是在动的，有些话到嘴边不记下来会溜走的，日子一天一天过去，生活也就不留在任何地方了。

我的默默无闻的技术理想如今已是进退两难，常常潜意识地思考要么是自己的努力或坚持程度不够，还不至于找天赋的借口，要么真是天生平庸，但我宁愿相信前者。

我与人沟通时不善于打情骂俏，容易直白表露自己的想法，这些沟通的障碍在互联网的技术时代越来越被社会所淘汰。互联网时代的技术人已不再需要独自埋头苦干了，他们善于各种娱乐调侃的段子，他们能直言不讳地评论政府社会，当然他们更善于开放地研究技术。

近些年来自己的一个变化是常常会陷于一些逻辑的思维中：万物有规律，技术有规律，社会有规律。人与人之间有平衡，有竞争，有合作。技术之间也有权衡，有取舍，有优化。

自己的种种缺陷都需要自己以更加开放的心态去解决，所以把自己最真实的想法记录下来，梳理自己，分享自己，希望若干年后暮然回首起那些失败或小有成就的岁月，你可以无悔地对它说拜拜了。

最后总结一下，希望这个Blog能带给自己：

  • 整理零碎的想法，形成系统的分析思考保险互联网产品的实验工场
  • Web技术的实验工场生活调味品网摘加评论
  • 给自己留下一份珍贵的电子财富吧。

现在又有个机会跟着李笑来学习Mac笔记本上计算机的编程，他推荐了一种学习计算机编程的方法，就是用假装已经成为高手，然后用上最新的技术来做事，这其中就举到建立博客网站的故事。

有必要说一下原来WordPress和Hexo写博客的不同点，WordPress提供一套PHP程序搭建一个博客网站，提供了写文章的后台管理界面，然后部署到服务器上，你没觉得写文章很麻烦吗？要打开界面把文章贴进去，然后发布。Hexo不是这个思路，写文章就像本地写Txt文本一样，然后它提供了自动化的发布功能。

做什么

利用Hexo在GitHub上建一个静态Blog站点。这点有三个概念：

  • Hexo
    Hexo是别人写的一个可以用来建立Blog站点并方便发布文章和管理站点的程序。

  • GitHub
    GitHub是一个网上的一个代码库。

  • Blog
    个人可以发表自己所写的文章。

准备工作

  • 在Github上上创建一个Repository，名称格式为mzxxgs.github.io
  • 事先安装好git和npm。git是代码库，npm是什么？

安装 Hexo

  1. 这一步不是很顺利，笑来网站是一种方法，官方网站也有安装方法，两个命令折腾半天，也不懂什么意思，国内下载也是个问题，网络环境也是问题，在家里网络环境试过，在单位网络环境试过，但折腾半天不知怎么还是给折腾上去了。
  2. 把Github上建立的Pepositoy克隆到本地
  3. 做一些初始化操作，这一步的目的很多软件要用起来就是要做一些初始化工作的，参见别人的文档或者官方文档照着做就行了，当然它每一步的操作背后都有很多细节的操作。
