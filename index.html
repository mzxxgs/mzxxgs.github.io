使用Hexo建立博客网站

很早之前就想建立自己的博客网站，因为发现写东西是多数牛人不断进步的共同方法，分享给别人其实是在梳理自己。所以也利用WordPress建立了一个网站，并且还在GoDaddy上申请了域名梦想照进现实,但是写了第一篇后就没有坚持下来，当时的那篇也可以转载过来。原因还是由于写不出来，写不出来的原因按照李笑来的说法有输入太少，没有处理能力。

现在又有个机会跟着李笑来学习Mac笔记本上计算机的编程，他推荐了一种学习计算机编程的方法，就是用假装已经成为高手，然后用上最新的技术来做事，这其中就举到建立博客网站的故事。

有必要说一下原来WordPress和Hexo写博客的不同点，WordPress提供一套PHP程序搭建一个博客网站，提供了写文章的后台管理界面，然后部署到服务器上，你没觉得写文章很麻烦吗？要打开界面把文章贴进去，然后发布。Hexo不是这个思路，写文章就像本地写Txt文本一样，然后它提供了自动化的发布功能。

做什么

利用Hexo在GitHub上建一个静态Blog站点。这点有三个概念：

  • Hexo
    Hexo是别人写的一个可以用来建立Blog站点并方便发布文章和管理站点的程序。

  • GitHub
    GitHub是一个网上的一个代码库。

  • Blog
    个人可以发表自己所写的文章。

准备工作

  • 在Github上上创建一个Repository，名称格式为mzxxgs.github.io
  • 事先安装好git和npm。git是代码库，npm是什么？

安装 Hexo

  1. 这一步不是很顺利，笑来网站是一种方法，官方网站也有安装方法，两个命令折腾半天，也不懂什么意思，国内下载也是个问题，网络环境也是问题，在家里网络环境试过，在单位网络环境试过，但折腾半天不知怎么还是给折腾上去了。
  2. 把Github上建立的Pepositoy克隆到本地
  3. 做一些初始化操作，这一步的目的很多软件要用起来就是要做一些初始化工作的，参见别人的文档或者官方文档照着做就行了，当然它每一步的操作背后都有很多细节的操作。

部署 Hexo

写好的文章要部署到服务器上用命令是多么简单的事情，而且命令要尽量简易。就是这么简单。
